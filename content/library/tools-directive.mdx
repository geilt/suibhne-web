A TOOLS.md file acts as a directive index. Instead of loading all configuration into context, it points to specific directive files that can be pulled when needed.

## The Pattern

```markdown
# TOOLS.md - Directive Index

Pull the relevant directive before acting. Don't load all of them.

---

## Directives

- `directives/messaging.md` â€” sending DMs on behalf of user
- `directives/databases.md` â€” MySQL, PostgreSQL, Redis
- `directives/calendar-email.md` â€” Gmail, Google Calendar
- `directives/browser.md` â€” browser profile, logged-in sites
```

## Why This Pattern?

### Context is Expensive

Loading every directive into every session wastes tokens. Most sessions only need 1-2 directives.

### Pull vs Push

**Push (bad):** Load everything at boot â†’ bloated context  
**Pull (good):** Index at boot, load specifics when needed

### Example Flow

1. User asks about Slack integration
2. Assistant sees `directives/messaging.md` in TOOLS.md
3. Assistant loads that specific file
4. Full Slack context available
